# 要約の長さを伸ばす

## 目的・背景

ダッシュボードの AI 要約が常に 140〜192 文字 (4 セッションでの実測) で頭打ちになっている。
原因は調査済みで、`ai-monitor/src/summarize.ts` のプロンプトで以下を明示しているため、
Haiku がその指示に素直に従っているため。

- `SYSTEM_PROMPT`: `"日本語 2〜3 行 (合計 200 文字程度) で要約"`
- `buildUserPrompt`: `"2〜3 行で要約してください"`
- `RESPONSE_MAX_TOKENS = 360` (≒日本語 180〜360 文字)

ツール往復が中心のセッションでも「実作業の証跡」が assistant-text に乗らない設計
(`tool-use` / `tool-result` は除外: Phase 1-F の確定方針) のため、要約は assistant
の地の文と user-text に依存する。短さを支配しているのはプロンプトの文字数指示。

## 対応方針

プロンプトでの文字数指示を緩めて、応答上限も合わせて引き上げる。入力側 (tool 除外 /
窓 / 1 行 400 字トリム) は触らない (確定方針 + 入力は現状 6000 字上限のうち 1800
字程度しか使われておらず、ボトルネックではない)。

具体変更 (`ai-monitor/src/summarize.ts`):

| 項目                  | before                      | after                              |
|---------------------|-----------------------------|------------------------------------|
| `SYSTEM_PROMPT`     | `2〜3 行 (合計 200 文字程度)`        | `4〜6 行 (合計 400〜600 文字程度)`            |
| user prompt 第 2 文   | `2〜3 行で要約してください`             | `4〜6 行で要約してください`                    |
| `RESPONSE_MAX_TOKENS` | `360`                     | `1000` (600 字を確実に書き切れる余裕)          |

## 影響範囲

- `ai-monitor/src/summarize.ts` のみ。`summarize.test.ts` は文字数指示の文言を
  アサートしていないので追加修正不要 (再確認済み)。
- 既存キャッシュは jsonl mtime 一致でヒットするため、サーバ再起動までは古い短い要約
  が一部残るが、jsonl が動けば次の `getOrCompute` で新仕様で再生成される。
- API コスト微増 (Haiku なので出力 600 字程度なら誤差レベル)。

## テスト方針

- 既存 `summarize.test.ts` (5 ケース) が通ること。
- 既存 `inspect-summary-input.mjs` / `probe-summary.mjs` で 3〜4 セッションの再要約
  を投げ、出力長が 400 字付近まで伸びることを確認。
